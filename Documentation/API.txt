Точки API с описанием функционала.
Прежде всего любая точка начивается с адреса "/api/<key>", где <key> - ключ. Он задаётся в файле settings.json 1 раз(в
модуле EEPROM есть для это соответсвующий метод), в пункте api_key. Все последующие точки начинаются с описанного ваше места.
Так же для всех точкек доступны исключительно GET запросы. В дальнейшем эта информация не будет повторяться.

>groups(предоставляет сведения обо всех группах в массиве объектов)
    Формат объекта: id, name
>group/<id>(предоставляет информацию о группе с конкретным id, указаным на месте <id> в виде объекта(смотри выше))
>group/<name>(предоставляет информацию о группе с конкретным именем, указаным на месте <name> в виде объекта(смотри выше))

>teachers(предоставляет сведения обо всех преподавателях в массиве объектов)
        Формат объекта: id, name
>teacher/<id>(предоставляет информацию о преподавателе с конкретным id, указаным на месте <id> в виде объекта(смотри выше))
>teacher/<name>(предоставляет информацию о преподавателе с конкретным именем(Фамилия И О), указаным на месте <name> в виде объекта(смотри выше))

>disciplines(предоставляет сведения обо всех дисциплинах в массиве объектов)
        Формат объекта: id, name
>discipline/<id>(предоставляет информацию о дисциплине с конкретным id, указаным на месте <id>  в виде объекта(смотри выше))
>discipline/<name>(предоставляет информацию о дисциплине с конкретным именем, указаным на месте <name> в виде объекта(смотри выше))


>subjects(предоставляет сведения обо всех занятиях(многие ко многим меджду преподавателми, группами, дисциплинами с
указанем положения на неделе, типа и аудитории+ признак изменения) в массиве объектов)
    Формат объекта: id, teacherId, groupId, disciplineId,  week, day, pair, type, place, diference
>subject/<id>(предоставляет информацию о занятии с конкретным id, указаным на месте <id> в виде объекта(смотри выше))
>group(предоставляет информацию о занятии по параметрам (применяются через оператор И ) в виде объекта(смотри выше))
    параметры:
        tid - id преподавателя
        gid - id группы
        did - id дисциплины
        w - неделя(0 - чётаня, 1 - не чётная)
        d - дкнь
        pn - пара в рамках дня
        t - тип (лек, пр, лаб)
        pl - место (аудитория) в виде строки
        m - признак модификации (0 - удалено, 1 - активно, 2 - обновелно)
    параметры применяются к запросу через ? и &

>timetable/<groupId>(предоставляет массив расписания для группы с id, указанным на месте <groupId>)
    каждая запись имеет вид: dname(название дисциплины), type(тип), tname(имя препподаателя), place(аудитория),
    modifired(признак модификации)

    При этом сама структура следующаяя(в скобках дано описание данного уровня массива с логической точки зрения):
        [
            [
                [
                    [
                        {Объект}(занятие в этот момент 1),
                        {Объект}(занятие в этот момент 2),
                        ...,
                        {Объект}(занятие в этот момент n)
                    ](1 пара),
                    [...](2 пара),
                    ...,
                    [...](8-ая пара)
                ](Понедельник),
                [...](Вторник),
                ...,
                [...](Суббота)
            ](чётная неделя),
            [...](нечётная неделя)
        ]
    В случае если в день нет ни одной пары - в его составе будут 8 пустых массивив-пар.
    Если на месте определённой пары - окно, то соответсвующий массив будет пустым.

ПРИМЕЧАНИЕ ОТ АВТОРА:
   -В качестве точки работы с системой настоятельно рекомендую timetable в связке с поиском по названию группы. Данный метод
    наиболее эффективен с точки зрения БД.
   -Для демонстрации возможностей на адресе /timetable/<groupName> (в данном случае это не часть api, тоесть указан прмямой путь)
    находится тестовая система вывода распиания в виде удобной(на мой взгляд) таблицы HTML. На место <groupName> нужно подставить навзание
    группы(правильное, тоесть правильно: "ПИбд-13", а не павильно "пибд-13", "пИбД-13", "ПИБД - 13" и тому подобное)
